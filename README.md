Необходимо создать программу учета заявок. Работать приложение должно в консоли. Всю разработку необходимо разбить на этапы.
Приложение должно осуществлять следующие действия.

Добавление новой заявки

Редактирования заявки.

Удаления заявки.

Отображение списка всех заявок.

Отображения заявки по идентификатору.

Поиск заявки по ключу (имени)

Добавлять к заявки комментарий.

В заявке должна быть следующая информация: Идентификаторб Имя, Описание, Дата создания, Автор И список комментариев. 
Кроме этого заявки могут быть трех разных типов.

Задания.

1) Нарисовать UML схему взаимодействия компонентов. Описать какие классы будут созданы. Сделать кратное описания, что должен делать каждый класс. Описать словами работу всего приложения. По возможности нарисовать блок схему основного цикла программы.

и Создать интерфейс для класса, который будет обрабатывать заявки.

2) Реализовать все классы и создать тесты для проверки их работы.
